## 0.1 2026-06-11 갱신 — 시그니처 전환 프로토타입 반영 (확정)

> 출처: 진입 전환 프로토타입을 구현·검증하며 확정한 디테일(프로토타입 코드는 세팅 시 제거 — 스펙은 본 문서에 보존).
> 아래는 본문 §2·§3·§5·§6 및 `design-system.md` §8을 **갱신/구체화**한다. 충돌 시 이 절을 따른다.

1. **라벨·메타 폰트 = Pretendard 대문자 스위스 라벨** (모노 폐기).
   - 기존 "라벨 = JetBrains Mono"(design-system §8.4)를 **변경**. `SELECTED PROJECTS` / `PM · UX · AI · FRONTEND` / `ROOM 01` 등 영문 라벨은 **Pretendard 600 + 대문자 + 넓은 자간(.18–.2em)**. 한글 라벨(역할 태그 등)은 대문자 없이 Pretendard 500.
   - JetBrains Mono는 진짜 코드성 텍스트(Tech Stack 토큰·prompt 스니펫)에만 선택적으로.
2. **Selected 블록(랜딩) = '무엇을 했는지' 한 줄 간단 설명** (라벨 템플릿 폐기): `문제/나의 판단/성과` 같은 라벨 구조 대신 프로젝트마다 **role 태그 + 한 줄 요약**으로 담백하게(스캔 우선). problem→판단→impact의 '닫힌 논증' 구조는 **상세 케이스스터디(planning §13)**로 이관 — 이중 동선(빠른 길=담백 / 느린 길=깊은 논증)과 정합. 데이터: `src/data/projects.ts`(role·summary).
3. **클릭 전 룸 색 정체성**: 각 Selected 블록에 ① accent-hue 옅은 룸 배경 ② 액센트 점 ● ③ '들어가기 →' CTA·프로젝트명을 그 색으로. **단 큰 제목은 검정 유지** → 클릭 시 검정→색 플립 임팩트 보존. 옅은 룸 톤: PULSE `#F1F5FC` · PickFit `#FBFDEA`(라임 계열) · LikeLion `#FFF5EE`.
4. **PickFit 라임 가독성 해결 = 동반색**: 라임 `#E3FF5D`는 흰 바탕 텍스트·선·마크에서 안 읽힘 → 텍스트·점·CTA·**플립색은 동반색 `#4D5EFF`(--pickfit-primary, 실제 브랜드 색)**. 라임은 옅은 룸 톤·포인트(채움)로만. (§8 가드레일의 'AI 블루/보라(LILA)'와 무관 — 그라데이션 아닌 브랜드 단색.) PULSE/LikeLion은 accent==기능색이라 변동 없음.
5. **중성 색 = design-system 공식 Slate 토큰** (임의 회색 금지): `--text-main #0F172A` / `--text-sub #475569` / `--text-muted #64748B` / `--line-soft #E2E8F0` (+ 더 흐린 라벨 `#94A3B8`).
6. **Avant Garde = typography_ref 로고타입 느낌**: 영문 디스플레이 전용, **대문자·볼드·자간 타이트(약 -0.04em)** — AVANT GARDE·NewJeans 워드마크 톤.
7. **Hero 워드마크 = `PORTFOLIO` / `NOH TAE KYUNG`** (대문자 Avant Garde 등폭 스택 락업). 사이트 **표시** 워드마크를 기존 "TAEKYUNG WORKS"에서 변경 — "TAEKYUNG WORKS"는 내부 폴더·문서 코드네임으로만 유지. 레이아웃 옵션 ⓐ PORTFOLIO 크게 + 이름 트래킹 바이라인(같은 폭) / ⓑ 자연 간격 바이라인. 서브 `PM · UX · AI · Frontend` 유지. (구현: §8.2.1 디자인된 레터링 기법).
8. **Avant Garde를 "디자인된 레터링"으로 취급** (단순 타이핑이 아님 — typography_ref가 진짜 원하는 것 = 글자가 타이포그래피로 디자인된 상태): 보유 폰트엔 **커스텀 합자 없음**(검증: 전 ttf가 `kern`만, `liga`/`dlig` 없음 → 합자 자동 불가). 대안 ① **라이브 근사** = 글자 겹침 + 각 글자에 **배경색 외곽선(notch)** → 같은 색 글자가 뭉개지지 않고 "맞물린 디자인"처럼 보임(겹침+notch 조절 방식). ② **로고급 천장** = 커스텀 벡터 레터링(획 공유·사선 인터록·광학 정렬, 워드마크당 1회, Figma/Illustrator 또는 손-SVG). 프로젝트 제목(PULSE 등)은 시그니처 플립 때문에 **라이브 타이트 텍스트 유지**. (방법 A/B 비교 검증 완료.)

## 1. 목적과 포지셔닝 (왜 이렇게 가는가)

- **1순위 = PM/서비스기획 취업.** 채용 담당자는 10초 스캔한다 → 스캔성이 생명.
- 동시에 **개성·완성도**로 기억에 남아야 한다(흔한 SaaS 포폴 탈피).
- 이 둘은 정반대로 당기므로 → **이중 동선**으로 해소:
  - **빠른 길 = 랜딩 페이지** (스캔러가 여기서 다 파악)
  - **느린 길 = 프로젝트 상세 페이지** (시간 있는 사람만 몰입)
- 하이브리드 포맷이 이 동선을 그대로 구현한다.

### 내러티브 척추 (포트폴리오를 관통하는 한 줄)
- 출처: 본인 벨로그 글 「AI가 90점을 준다고 끝난 게 아니었다」(너진똑 리뷰).
- 방향(확정): **"AI가 90점을 내도록 환경을 설계하고, 마지막 10점은 직접 해본 경험과 판단으로 채운다. 나는 사장이고 AI는 만능 직원이다."**
- 강력한 이유 = **메타 증거**: 이 포트폴리오·벨로그·pm-teacher 스킬·PULSE/PickFit을 전부 이 방식으로 본인이 직접 만들었다. 매체 자체가 메시지.
- ✅ **톤 확정 = 톤1(담백·전문):** "AI가 90점을 만드는 시대. 나는 그 90점을 더 잘 뽑아내고, 마지막 10점을 직접 해본 경험으로 채웁니다."
  - 대체 톤(필요 시 본문/서브 카피에 변주): (단언) "AI는 90점까지. 마지막 10점, 그게 제가 하는 일입니다." / (비유) "저는 사장이고, AI는 만능 직원 — 결정은 제가 합니다."

## 2. 비주얼 언어 & 위계 (가장 중요한 결정)

자산엔 시각 언어가 3개 떠다닌다 → **하나를 보스로** 정해 일관성 확보.

| 레이어 | 언어 | 역할 |
|--------|------|------|
| **골격 (랜딩·케이스 본문)** | 미니멀 스위스 타이포 | 스캔성·가독성·채용 신뢰. 흰 바탕 + 큰 타이포 + 페이지별 액센트 1색 |
| **몰입 액센트 (Hero·상세 룸)** | 종이 콜라주 + 2.5D 갤러리 | corridor=방 입구 전환, room bg=상세페이지 백드롭, 오브젝트=스크롤 플로팅 |
| **벨로그 섹션 전용** | MS Paint 낙서 | 날것의 사고 공간 — 의도적 대비로 개성 |

- **원칙:** 콜라주는 "메인 무대"가 아니라 "타이포 골격 위의 개성 양념".
- 이렇게 하면 자산 투자도 살리고, 채용 친화 스캔성·절제도 얻는다.

## 3. 컬러 규칙 (블루 과다 문제 해결)

생성 자산이 전부 블루 지배 → design-taste의 **"AI 블루(LILA) = AI생성 SaaS 느낌"** 경고에 정확히 걸림. 해결:

- **베이스는 흰색/검정 타이포로 전 페이지 동일.**
- **딱 한 색만 그 프로젝트 액센트로** 켠다:
  - PULSE = 딥블루 `#002B7A` (차분·전문)
  - PickFit = 형광 라임 `#E3FF5D` (발랄·패션)
  - LikeLion = 오렌지 `#FF6000` (따뜻·교육)
- 페이지를 넘기면 **흰 바탕은 그대로, 포인트 색만 확 바뀌어** 방이 바뀐 게 또렷.
- → "색이 다른 전시 방"이 콜라주 배경이 아니라 **타이포+액센트**로 살아난다.
- **규칙 한 줄:** "흰 골격 위에서, 색은 의미를 가질 때만 켜진다."

### Hero & 10초 요약 (✅ 확정 카피·구성)

**Hero** (흰 화면, 왼쪽 정렬 + 우측 콜라주 자화상)
- 좌상단 라벨(모노): `노태경 — PM · UX · AI · Frontend`
- ✅ **첫 문장 = 헤드라인(Avant Garde, 검정):** "AI가 답을 만드는 시대에, 저는 마지막 판단을 설계합니다."
- 서브라인(Pretendard, 회색): "복잡한 문제를 바로 기능으로 바꾸지 않고, 사용자가 결정할 수 있는 흐름으로 만듭니다."
- 우측: 폰 보는 콜라주 자화상 + 파란 메모지 (마우스엔 메모지·커서만 살짝 반응, 타이포·인물은 고정).
- 등장: 헤드라인 2줄 stagger-up → 서브라인 → 자화상 → 오브젝트 톡톡.

**10초 요약** (Hero 바로 아래, 왼쪽 정렬, **박스 없이** 타이포 + 얇은 선 + 색점)

- **대표 작업 3줄** (카드 아님, 앞에 액센트 점):

## 6. 시그니처 인터랙션 — 프로젝트 진입 전환 (redcollar.co식)
> ✅ 2026-06-11 갱신(§0.1): 플립 뒷면색=액센트는 유지하되 **PickFit만 라임 비가독으로 동반색 `#4D5EFF`**. 톤(원확산 커버)은 아래 값 유지. 프로토타입에서 타이밍·이징 검증 완료(코드 제거, 스펙은 §6 보존).

### 레퍼런스 분석 결과 (Playwright로 redcollar.co 직접 구동·분석)
- redcollar는 **통짜 커스텀 WebGL**: `<canvas>` 3개, 제목이 `<div class="project-mesh" data-src="….obj">` (3D 모델 얹은 WebGL 메시), `<page-transition><div class="overlay">` 전용 전환 레이어.
- 즉 "글자 회전 + 색변화 + 배경 원확산"은 **셰이더(WebGL)로 만든 효과**.

### 효과 묘사 (눈앞에 그려지게)
1. **글자 뒤로 한 바퀴 + 색 변화**: 까만 제목을 클릭하면, 글자 윗변이 뒤로 넘어가며 가로축으로 **한 바퀴 공중제비**(옛 공항 안내판 한 칸 넘듯). 도는 동안 색이 검정→액센트색으로 물들고, 첫 글자가 살짝 먼저·끝 글자가 살짝 늦게 돌아 **왼→오른쪽 물결**. 약 0.7초.
2. **배경 원확산**: 같은 순간 화면 **정중앙에 연한 색 원**이 태어나 사방으로 부풀어 화면을 덮는다. 흰 종이에 옅은 잉크 한 방울이 퍼지듯, 배경이 흰색→그 프로젝트 연한 톤으로. 약 0.6~0.7초.
3. 둘이 동시에 터지고 → 그 색이 화면을 덮은 채 상세 페이지 안착.

### 정밀 스펙 (✅ 확정 · 서브에이전트 검토 반영)
> 핵심 교정: 360° 평면 플립은 0.7초간 제목이 안 읽히는 "한 바퀴 깜빡임" → **CSS 3D 2면 플립카드(0→180°)**로 교체. WebGL 텍스트는 폰트 충실도 손실이라 **전환은 CSS+GSAP, WebGL은 Hero에만**. "퀄리티 비타협" 기준에서 CSS가 오히려 더 고품질.

- **트리거 단일화:** Selected "들어가기 →"에서만(10초요약은 앵커). → "진입 = 비싸고 드문 의식".
- **① 글자 플립 = CSS 3D 2면 플립카드:** 각 글자 = 앞면(검정)/뒷면(액센트색) 카드. `rotateX 0→180°`, **0.5~0.55s**, stagger 30ms(**클릭한 글자부터** 시작), ease `expo.out(.16,1,.3,1)`, `perspective:800px` + `preserve-3d`. → 면 교체로 **검정→액센트가 또렷이** 갈림(깜빡임·가독공백 0, 색 트윈 불필요).
- **② 배경 원확산:** ①이 먼저 읽힌 뒤(0~0.4s) → 연한 톤 오버레이가 `clip-path: circle(0%→150% at 50% 50%)` **중심 고정**, **0.3s~0.85s**, ease `(.76,0,.24,1)`. 톤: PULSE `#EAF1FB` · PickFit `#E9ECFF` · LikeLion `#E6F2FF`.
- **순서 = 읽고(변색·플립) → 덮기(확산).** 동시 폭발 금지(제목 정체성 보호). 클릭 위치는 확산 원점이 아니라 **플립 stagger 시작점**으로만 반영.
- **커버 persist → 흰 플래시 차단:** 라우트 교체 동안 연한 톤 커버 유지, 상세 룸 **첫 프레임 ready 신호**에 RoomEntrance fade-up(0.4s) 게이트.
- **총 ~0.9s**(반복 진입 피로 완화). 이미 본 방 재진입은 더 짧게(옵션).
- **뒤로가기:** 역재생 대신 커버 **0.3s 크로스페이드**로 랜딩 복귀(역재생은 언마운트·스크롤복원 충돌로 비용↑).
- **모바일·저성능·reduced-motion:** 자동 경량 = **0.25~0.3s 크로스페이드 + 제목 색만 변경**(채용자 다수가 모바일 첫 스캔).
- **일석이조:** 이 전환이 "방마다 색이 달라지는 법"과 "redcollar 효과"를 한 동작에서 동시 해결.
